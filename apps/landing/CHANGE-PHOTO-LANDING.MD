# Замена фото на лендинге «Я на стиле»

Инструкция: куда положить файлы и что править в коде, чтобы на сайте появились ваши реальные изображения.

---

## Быстрый старт

1. Положите файлы в папку **`public/assets/`** (путь в браузере: `/assets/имя-файла.png`).
2. Откройте **`content/image-slots.ts`** — там все слоты лендинга в одном месте.
3. Для каждого слота укажите свой `src` (и при желании обновите `alt`).
4. Перезапустите dev-сервер или пересоберите проект: `npm run dev` / `npm run build`.
5. Обновите страницу в браузере с **Ctrl+F5**.

**Важно:** менять пути в компонентах (`HomePage.tsx`, `seo-pages.ts` и т.д.) обычно **не нужно** — они берут картинки из `image-slots.ts`.

---

## Главный конфиг

| Файл | Назначение |
|------|------------|
| `content/image-slots.ts` | Все ключевые изображения лендинга и SEO-страниц |
| `content/seo-pages.ts` | Какие слоты на какой URL (через `imageSlots.*`) |
| `components/home/HomePage.tsx` | Главная страница `/` |
| `app/layout.tsx`, `lib/seo.ts` | Превью для соцсетей (Open Graph) |

### Макияж (отдельные close-up)

Рекомендуемая папка: **`public/assets/makeup/`**

   - `public/assets/makeup/evening-before.jpg`

```ts

```

**Совет:** для «до/после» — один ракурс, одно освещение, крупный план лица.

## Как сайт покажет фото, если размер файла другой

**Запомнить перед вставкой:** в оригинальном размере в пикселях картинки на сайте **не выводятся**. Размер на экране задаёт вёрстка (CSS), а не файл. Большой файл ≠ большая картинка на странице (влияет только на качество и вес).

### Сжатие и пропорции

| Вопрос | Ответ |
|--------|--------|
| Сожмётся ли «как мыло» по ширине/высоте по-разному? | **Нет.** Пропорции всегда сохраняются. |
| Уменьшится ли файл для загрузки? | **Да.** Next.js отдаёт оптимизированную версию (WebP и т.д.) — это не искажает форму на экране. |
| Влезет ли всё фото целиком? | **Не всегда.** Во многих блоках лишнее **обрезается** (см. ниже). |

### Три режима на лендинге

| Режим | Где | Пропорции | Обрезка |
|--------|-----|-----------|---------|
| **`object-fit: cover`** | Большинство блоков | Сохраняются | **Да** — края |
| **Ширина 100%, высота `auto`** | Коллаж в hero, телефоны, CTA | Сохраняются | **Нет** — видно всё фото |
| **`fill` + `cover`** | Мозаика `/podbor-obraza` | Сохраняются | **Да** — края (кадр с верха) |

### Если обрезается (`cover`) — что отрежется

Картинка **равномерно увеличивается**, пока не заполнит рамку блока. Что не влезло — срезается.

**По умолчанию** точка кадра — **центр** (`object-position: center`).

| Ваше фото | Рамка на сайте | Что обрежется |
|-----------|----------------|---------------|
| Вертикальное (портрет) | Широкая, низкая | **Слева и справа** |
| Горизонтальное (альбом) | Высокая, узкая | **Сверху и снизу** |
| Пропорции близки к рамке | — | Почти ничего |

**Исключение — `/podbor-obraza` (мозаика 2×2):**  
`object-position: center top` — якорь **верх по центру**. Для портрета чаще видны **лицо и верх образа**, срезается **низ** (и при необходимости бока). Высота плитки в CSS: **46vh**.

### Поведение по блокам (что ждать при подстановке)

#### Главная `/`

| Блок | Слот | Обрезка? | Как кадрируется |
|------|------|----------|-----------------|
| Hero слева — «ты без образа» | `heroBefore` | Да | Рамка ~590px по высоте, **центр** |
| Hero справа — коллаж | `heroCollage` | **Нет** | Вся картинка по ширине, высота по пропорциям |
| «До и после» | `beforeAfter` | Да | Высота ~330px, **центр** |
| «Подходит всем стилям» | `styles` | Да | Высота ~330px, **центр** |
| Галерея 4 карточки | `femaleCard1`…`4` | Да | Высота 280px, **центр** |
| Телефоны | `phoneMockups` | **Нет** | Пропорции файла |
| CTA сумки + QR | `ctaBags`, `qrDemo` | **Нет** | Масштаб по ширине / 150px |

#### SEO-страницы

| Блок | Обрезка? | Как кадрируется |
|------|----------|-----------------|
| Hero (например `/primerka-platya`) | Частично | Ширина 100%, max-height 480px, `cover`, **центр** |
| Split `/ai-primerka` | Да | Как «До и после», ~330px, **центр** |
| Мозаика `/podbor-obraza` | Да | Рамка 46vh, `cover`, **верх + центр** |
| До/после `/makiyazh` | Да | Высота ~380px, **центр** |
| Мозаика `/pricheski` (2 фото) | Да | Высота 360px на плитку, **центр** |

На очень узком экране у блоков «До и после» / `showcase-img` высота может стать `auto` — тогда обрезки нет, только масштаб по ширине.

### Как готовить файлы под обрезку

1. **Портреты** (`female-card-*`, hero «до», плитки подбора образа) — ориентир **3:4**, важное **по центру**; для лица — чуть **выше центра** (особенно для `/podbor-obraza`).
2. **Широкие баннеры** (`before-after`, `styles`) — **горизонталь** 4:3 или 3:2, важное **в центре кадра**.
3. **Коллаж hero** — можно шире; обрезки нет, но слишком «высокий» файл удлинит страницу по вертикали.
4. **Макияж до/после** — один ракурс, одно освещение; лицо **по центру**, иначе при обрезке «уедет» половина лица.

### Если нужно показать фото целиком без обрезки

Сейчас так сделано не везде — по умолчанию дизайн с `cover`. Чтобы в конкретном блоке было «всё видно» (с полями по краям), нужно менять вёрстку (`object-fit: contain` или другая высота рамки). Напишите, для какого блока — настроим отдельно.

## Если картинка не обновилась

1. Убедитесь, что файл именно в **`public/`**, а не в `content/` или корне проекта.
2. Путь в `src` начинается с **`/assets/`** (слэш в начале обязателен).
3. Имя файла в `src` **совпадает** с именем на диске (регистр букв на Linux-сервере важен).
4. Очистите кэш браузера (Ctrl+F5).
5. При деплое — пересоберите проект (`npm run build`).
