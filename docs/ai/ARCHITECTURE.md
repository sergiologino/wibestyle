# Architecture

## Текущая целевая архитектура (MVP)
- Формат: один frontend-сервис.
- Тип: Next.js (App Router), TypeScript.
- Назначение: многостраничный SEO-ориентированный лендинг с rich visual UI.

## Планируемые модули
- `app/` — маршруты и страницы (главная + SEO-подстраницы).
- `components/` — UI-секции и переиспользуемые блоки.
- `lib/` / `content/` — SEO-конфиг, метаданные, контентные структуры.
- `public/images/` — плейсхолдеры и будущие реальные медиа.
- `docs/ai/` — единственный источник памяти проекта.

## Интеграции
- На этапе MVP backend обязателен только при необходимости обработки лидов.
- Стартово: форма может работать без внешнего API (локально/заглушка), с готовностью к подключению backend позже.

## Масштабирование
- При росте требований возможно расширение до монорепозитория:
  - frontend (landing/app),
  - backend leads/api,
  - optional analytics service.
