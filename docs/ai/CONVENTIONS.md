# Conventions

## Общие правила работы
- Память проекта хранится только в `docs/ai/`.
- История чата не используется как долговременная память.
- Любые кодовые изменения сопровождаются обновлением `CURRENT_STATE.md`.
- По завершении задачи добавляется запись в `CHANGELOG_AI.md`.

## Инженерные правила
- Изменения вносить минимально инвазивно.
- Не ломать рабочий функционал без явного запроса.
- Избегать массовых рефакторингов без необходимости.
- Архитектурные решения фиксировать в `DECISIONS.md`.
- Если решение признано неверным: пометить как Superseded + синхронизировать код и память.

## Тестирование
- На каждую фичу добавлять автотесты.
- Тесты должны запускаться в сборочном пайплайне.
- При падении теста: найти причину, исправить, повторить сборку.

## Контент memory-файлов
- Допускается только итоговое знание.
- Документация по доменам: [API.md](./API.md), [PROMO_CODES_GUIDE.md](./PROMO_CODES_GUIDE.md).
- Запрещено хранить:
  - логи,
  - stacktrace,
  - временные рассуждения,
  - промежуточные гипотезы.

## Стиль реализации (для текущего продукта)
- Визуальный приоритет: image-first, fashion-tech, плотная композиция.
- Контент-приоритет: SEO/GEO-дружественный, индексируемый HTML.
